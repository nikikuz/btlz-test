# btlz-test

## Описание

**btlz-test** — это тестовое приложение для работы с PostgreSQL и API Wildberries. Включает в себя миграции, сиды и периодические задачи с использованием cron для взаимодействия с API.

## Структура проекта

- **Knex.js** для работы с базой данных.
- **Node-Cron** для периодических вызовов API (например, получения списка складов с Wildberries).
- **Docker** для упрощения развертывания приложения.

## Установка и запуск

Для начала работы с проектом выполните следующие шаги:

### 1. Клонируйте репозиторий

```bash
git clone https://your-repository-url.git
cd btlz-test
```

### 2. Создайте файл .env
Пример .env файла:
```.env
POSTGRES_DB=root
POSTGRES_USER=root
POSTGRES_PASSWORD=root
APP_PORT=5000
WB_TOKEN=ваш_токен_с_вильдберриз
WB_API_URL=https://common-api.wildberries.ru/api/v1/tariffs/box
```

Обратите внимание, что вам нужно будет создать файл .env с вышеуказанными переменными, чтобы настроить подключение к базе данных и API.

### 3. Соберите и запустите контейнеры с помощью Docker Compose
Для того чтобы собрать и запустить проект с Docker, используйте команду:

```bash
docker-compose up --build
```

Эта команда автоматически:
Соберет образ для вашего приложения.
Поднимет контейнеры для PostgreSQL и вашего Node.js приложения.
Выполнит все миграции и сиды.
Настроит задачу cron для вызова API каждый час.
4. Что будет происходить?
Миграции и сиды: При запуске приложение выполнит все миграции и сиды базы данных.
Периодический вызов API: Каждый час будет выполняться запрос к API Wildberries для получения актуальных данных о складах.

### 5. Структура файлов
docker-compose.yml: Конфигурация для Docker.
Dockerfile: Образ для запуска Node.js приложения.
src: Исходный код проекта.
postgres: Миграции и сиды для базы данных.
services: Логика для работы с API Wildberries.
.env: Файл с конфигурационными переменными.
### 6. Пример .env файла
```.env
POSTGRES_DB=root
POSTGRES_USER=root
POSTGRES_PASSWORD=root
APP_PORT=5000
WB_TOKEN=ваш_токен_с_вильдберриз
WB_API_URL=https://common-api.wildberries.ru/api/v1/tariffs/box
```

Не забудьте заменить ваш_токен_с_вильдберриз на ваш реальный токен.

### 7. Просмотр логов приложения
После того как контейнеры будут запущены, вы можете следить за логами приложения с помощью команды:

```bash 
docker-compose logs -f
```

### 8. Остановка приложения
Чтобы остановить все контейнеры, используйте:

```
bash
docker-compose down
```

### 9. Примечания

Убедитесь, что Docker и Docker Compose установлены на вашем компьютере.
Для локальной разработки вы можете использовать команду npm start после того как соберете проект.
Чтобы добавить новые миграции или сиды, просто добавьте новые файлы в папки src/postgres/migrations и src/postgres/seeds и выполните миграцию заново с помощью knex migrate:latest.
